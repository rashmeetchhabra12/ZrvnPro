# 🚀 Deployment Guide

This document provides step-by-step instructions for deploying the Finance Dashboard Backend to various cloud platforms.

## 📋 Prerequisites

- GitHub account
- Node.js 18+ installed locally
- PostgreSQL database (will be provided by cloud platforms)

## 🌐 Platform Options

### 1. Railway (Recommended) 🚂

**Pros:** Simple PostgreSQL setup, great free tier, excellent Node.js support

**Steps:**
1. Sign up at [railway.app](https://railway.app) with your GitHub account
2. Click "New Project" → "Deploy from GitHub repo"
3. Select your `finance-dashboard-backend` repository
4. Railway will detect the `railway.toml` configuration automatically
5. Add a PostgreSQL database:
   - Go to your project dashboard
   - Click "New" → "Database" → "PostgreSQL"
   - Railway will auto-generate DATABASE_URL
6. Set environment variables:
   ```
   NODE_ENV=production
   PORT=3000
   DATABASE_URL=[auto-generated by Railway]
   ```
7. Deploy will happen automatically!

### 2. Render 🎨

**Pros:** Free PostgreSQL database, automatic GitHub deploys

**Steps:**
1. Sign up at [render.com](https://render.com) with your GitHub account
2. Connect your GitHub repository
3. Render will detect the `render.yaml` configuration
4. The PostgreSQL database will be created automatically
5. Environment variables are configured in the YAML file
6. Deploy will start automatically!

### 3. Heroku ⚡

**Pros:** Well-established platform, lots of documentation

**Steps:**
1. Install [Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli)
2. Login: `heroku login`
3. Create app: `heroku create your-app-name`
4. Add PostgreSQL: `heroku addons:create heroku-postgresql:mini`
5. Set environment variables:
   ```bash
   heroku config:set NODE_ENV=production
   ```
6. Deploy: `git push heroku main`

## 📊 Environment Variables

All platforms need these environment variables:

```env
NODE_ENV=production
DATABASE_URL=[provided by platform]
PORT=[provided by platform]
```

## 🧪 Post-Deployment Testing

After deployment, test these endpoints:

```bash
# Health check
curl https://your-app-url.com/health

# API documentation
curl https://your-app-url.com/api-docs

# Test login (will create test data)
curl -X POST https://your-app-url.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password123"}'
```

## 🔍 Troubleshooting

### Common Issues:

**Database Connection Errors:**
- Verify DATABASE_URL environment variable
- Check if database is running
- Ensure database security settings allow connections

**Port Issues:**
- Use the PORT environment variable provided by the platform
- Don't hardcode port 3000 in production

**Build Failures:**
- Check Node.js version (should be 18+)
- Verify all dependencies are in package.json
- Review build logs for specific errors
